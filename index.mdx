Trace-First Workflow (TFW) makes AI-assisted work permanent. Every task produces structured trace artifacts — HL, RES, TS, ONB, RF, REVIEW — that record not just *what* was built, but *why*, *what was considered*, and *what was rejected*. When a chat session ends or a team member changes, the project doesn't reset. The next agent reads the traces and picks up exactly where the previous one left off.

## How TFW works

TFW is a structured ritual. Every task moves through a deterministic pipeline. Each stage produces a specific artifact stored as a file in your project. Those files are your project's permanent memory — readable by any AI agent or team member in any future session.

<Steps>
  <Step title="Plan the task">
    Run `/tfw-plan` to create a High Level artifact (HL) — the map of intent, constraints, and decisions before any implementation begins.
  </Step>
  <Step title="Research (optional)">
    Run `/tfw-research` for structured investigation. The Researcher role follows an OODA loop through Gather, Extract, and Challenge stages, producing a RES synthesis artifact.
  </Step>
  <Step title="Specify and execute">
    The Coordinator writes a Task Spec (TS) with scope, steps, and acceptance criteria. Run `/tfw-handoff` to give the Executor a structured onboarding report before implementation begins.
  </Step>
  <Step title="Review and close">
    Run `/tfw-review` for a formal 9-point checklist review. Approved tasks flow to knowledge capture — Fact Candidates are consolidated into verified project knowledge that compounds over time.
  </Step>
</Steps>
