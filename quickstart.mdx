TFW requires no installation — it's a methodology, not a package. Setup means copying a single prompt to your AI agent (Claude Code, Cursor, or any chat). The agent clones the starter, reads the conventions, and initializes the project for you.

## Choose your starting point

<Tabs>
  <Tab title="New project">
    Copy this prompt into your AI agent:

    ```text
    I want to start a new project using Trace-First Workflow (TFW) —
    a methodology that preserves decisions, reasoning, and knowledge across AI sessions.
    Clone https://github.com/saubakirov/trace-first-starter to a temp directory,
    then read .tfw/quickstart.md and follow it step by step.
    My project is about: <describe your project in a few sentences>
    ```
  </Tab>
  <Tab title="Existing project">
    Copy this prompt into your AI agent:

    ```text

    Clone https://github.com/saubakirov/trace-first-starter to a temp directory,
    copy the .tfw/ directory into my project root, then delete the temp clone.
    Then read .tfw/quickstart.md and follow it step by step.
    My project is about: <describe your project>
    ```
  </Tab>

## What happens during initialization

<Steps>
  <Step title="Agent clones the starter">
    Your AI agent clones `https://github.com/saubakirov/trace-first-starter` and copies the `.tfw/` directory into your project root. This is the tool-agnostic TFW core.
  </Step>
  <Step title="Agent reads the conventions">
    The agent reads `.tfw/quickstart.md`, `.tfw/README.md`, `.tfw/glossary.md`, and `.tfw/conventions.md` in order to understand the lifecycle, roles, and artifact types.
  </Step>
  <Step title="Agent runs /tfw-init">
    The initialization workflow (`/tfw-init`) guides the agent through project discovery — it interviews you, sets up `AGENTS.md`, `KNOWLEDGE.md`, `README.md` with a Task Board, and configures `.tfw/PROJECT_CONFIG.yaml` for your project.
  </Step>
  <Step title="Agent sets up your tool adapter">
    Based on your tool (Claude Code, Cursor, Antigravity), the agent installs the appropriate adapter so TFW slash commands are available in your environment.
  </Step>
  <Step title="Start your first task">
    Run `/tfw-plan` with a task description. The Coordinator creates an HL artifact, optionally runs research, writes a Task Spec, and hands off to execution.
  </Step>
</Steps>

## Slash commands reference

Once initialized, these commands drive the entire TFW lifecycle:

| Command | Role | Purpose |
|---------|------|---------|
| `/tfw-plan` | Coordinator | Create HL + TS for a new task |
| `/tfw-research` | Researcher | Run structured investigation, produce RES artifact |
| `/tfw-handoff` | Executor | Onboard, implement, write RF Result File |
| `/tfw-review` | Reviewer | 9-point checklist review, issue APPROVE/REVISE/REJECT verdict |
| `/tfw-resume` | Coordinator | Resume interrupted work from any checkpoint |
| `/tfw-knowledge` | Coordinator | Consolidate Fact Candidates into verified knowledge |
| `/tfw-docs` | Coordinator | Update KNOWLEDGE.md and TECH_DEBT.md after task close |
| `/tfw-release` | Coordinator | Cut a versioned release with changelog |
| `/tfw-update` | Coordinator | Upgrade `.tfw/` from upstream starter |
| `/tfw-init` | Coordinator | Initialize TFW in a new or existing project |
| `/tfw-config` | Coordinator | Change config and propagate to inline values |

<Note>
  You only need to read `.tfw/README.md` — the philosophy takes about 5 minutes. Your AI agent handles everything else.
</Note>

## Frequently asked questions

<AccordionGroup>
  <Accordion title="Do I need to read the documentation?">
    No. The `.tfw/` files are designed for AI agents. You only need the [philosophy](/philosophy) — it explains why TFW works and takes about 5 minutes.
  </Accordion>
  <Accordion title="Which AI tools work with TFW?">
    Any tool that can read files. Adapters exist for Claude Code, Cursor, and Antigravity — your agent sets them up during init. For plain chat (ChatGPT, etc.), have the agent read `.tfw/README.md` directly.
  </Accordion>
  <Accordion title="Can I use TFW for non-code work?">
    Yes. TFW is about structuring decisions, not writing code. The same lifecycle works for analytics, writing, education, business processes, and academic research.
  </Accordion>
  <Accordion title="How is TFW different from Confluence or Notion?">
    Confluence and Notion are knowledge *storage* tools — they hold what someone decides to write. TFW is a knowledge *generation* methodology — it captures decisions, reasoning, and alternatives as a byproduct of working. You don't document your decisions; your decisions document themselves.
  </Accordion>
</AccordionGroup>
