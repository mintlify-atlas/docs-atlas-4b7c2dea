TFW uses [Semantic Versioning](https://semver.org/): `MAJOR.MINOR.PATCH`. The current installed version is recorded in `.tfw/VERSION`.

## Checking your version

```bash
cat .tfw/VERSION
```

<Note>
To upgrade your project to the latest TFW version, run `/tfw-update`. The update workflow fetches the upstream starter, compares versions, categorizes changes, and walks you through a checklist to re-sync your adapters and inline values.
</Note>

- **BREAKING:** HL template §1 restructured — generic "Vision" → Vision narrative + Impact + Quote

## [0.6.6] — 2026-04-04

## Versioning scheme

TFW uses [Semantic Versioning](https://semver.org/):

| Segment | Meaning |
|---|---|
| `MAJOR` | Incompatible changes to artifact formats or workflow protocol |
| `MINOR` | New workflows, new config fields, new artifact types — backward compatible |
| `PATCH` | Bug fixes, documentation corrections, adapter sync, template wording |
